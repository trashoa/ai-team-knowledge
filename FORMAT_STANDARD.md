## 📝 知识条目模板

### 基础信息（YAML 头）

```yaml
---
id: SEC-001
category: 安全漏洞
severity: critical | high | medium | low
tags: [关键词1, 关键词2, 关键词3]
created: 2026-03-14
updated: 2026-03-14
author: 大黑手
related_issues: [35, 39]
status: active | deprecated
---
```

### 内容结构

```markdown
# 标题（问题名称）

## 问题描述
清晰简洁地描述这个问题是什么。

## 影响范围
这个问题会造成什么影响？

## 错误示例
展示错误的代码示例。

## 正确做法
展示正确的代码示例。

## 检测方法
如何检测这个问题？（工具、方法）

## 修复方案
具体的修复步骤。

## 参考资料
相关的文档、链接。
```

## 🏷️ 分类标签

### 按严重程度 (severity)

| 标签 | 说明 |
|------|------|
| critical | 严重，必须立即修复 |
| high | 高危，建议尽快修复 |
| medium | 中等，常规修复 |
| low | 低危，可选修复 |

### 按分类 (category)

| 分类 | 前缀 | 说明 |
|------|------|------|
| 安全漏洞 | SEC | 安全相关问题 |
| 审查规则 | RULE | 代码风格、检查规则 |
| 修复方案 | FIX | 具体修复方法 |
| 常见问题 | FAQ | 常见问题解答 |

### 按状态 (status)

| 状态 | 说明 |
|------|------|
| active | 有效 |
| deprecated | 已过时 |

## 🔖 入库标签（更新）

### 标记格式

```
[建议入库：分类]
```

### 分类标签

| 标签 | 用途 |
|------|------|
| `[建议入库：安全漏洞]` | 硬编码密码、SQL注入、XSS等 |
| `[建议入库：审查规则]` | 评分标准、检查清单 |
| `[建议入库：修复方案]` | WebSocket认证、输入验证等 |
| `[建议入库：常见问题]` | 配置问题、环境问题 |

### 标记时机

**✅ 应该标记：**
- 发现新的安全问题类型
- 发现重复出现的问题
- 发现好的修复方案

**❌ 不用标记：**
- 一次性配置错误
- 已有类似知识
- 项目特定问题

## 🤖 AI 检索逻辑

### 检索流程

```
1. 大黑手审查代码
   ↓
2. 发现问题类型（如硬编码 Token）
   ↓
3. 根据类型检索知识库（如 SEC-001）
   ↓
4. 找到相关知识
   ↓
5. 注入提示词（如"检查硬编码密码"）
   ↓
6. 提升审查准确性
```

### 示例

发现硬编码 Token → 检索 SEC-001 → 注入提示词 → 识别更准确

## 🔄 知识更新机制

### 维护规则

| 操作 | 处理方式 |
|------|----------|
| 新知识 | 直接添加 |
| 旧知识 | 添加 `status: deprecated` 标记 |
| 重复知识 | 合并到现有条目 |
| 错误知识 | 修正并记录更新日志 |

### Review 频率

- **频率**: 每月最后一个周五
- **负责人**: 光枢 + 大黑手
- **内容**: 检查过时知识、合并重复条目

## 📋 入库检查清单

入库前检查：

- [ ] id 唯一（格式：类型-序号，如 SEC-001）
- [ ] category 正确（安全漏洞/审查规则/修复方案/常见问题）
- [ ] severity 合理（critical/high/medium/low）
- [ ] tags 完整（至少3个）
- [ ] created 日期正确
- [ ] related_issues 填写
- [ ] 内容结构完整（问题描述/错误示例/正确做法/检测方法/修复方案）
- [ ] 代码示例已验证
